## 浏览页面批量接口性能优化

**问题**：原实现对每张卡片逐条调用 `Note.get()`，每次全量扫描 notes 数组（O(N×M)），卡片多时明显卡顿

**服务端新增接口**（`server.js`）：
- `GET /api/browse/tree`：一次返回所有 decks、folders、及每个 deck 的统计（new/learn/review/total），通过 cards 一次遍历完成
- `GET /api/browse/deck/:id`：一次返回指定 deck 的所有 cards 和对应 notesMap（key=nid）

**前端优化**（`anki.html`）：
- `_renderBrowseTree` 改为一次 fetch `/api/browse/tree`，渲染完全同步（无异步循环）
- `_renderBrowseDeckCards` 改为一次 fetch `/api/browse/deck/:id`，数据缓存在 `_browseDeckCache`
- `renderBrowseDeckList` 改为纯同步函数（从 notesMap 直接查，不再 await）
- `browseDeckSearch` 完全基于内存缓存过滤，零额外网络请求

## 卡组视图分页渲染（防卡顿）

**问题**：即便数据加载快了，一次性渲染几千个 DOM 节点仍会让浏览器卡死

**实现**：
- 每批渲染 `_browsePageSize = 50` 条，底部放 sentinel 元素
- `IntersectionObserver` 监测滑动到底部（`rootMargin: 200px` 预加载）自动调用 `_browseAppendCards()`
- 用 `insertAdjacentHTML('beforeend')` 追加，不重绘已有 DOM
- 搜索时重置为从头加载；删除后重新进入时重置
- 变量：`_browseCurrentCards`、`_browseCurrentNotesMap`、`_browseLoadedCount`、`_browseObserver`、`_browseSearchMode`
