# Anki 快速制卡 Web 项目 开发需求报告

## 一、项目核心定位
一款**纯前端轻量级 Web 应用**，无需安装、无需服务器/数据库，浏览器打开即用；核心能力是**可视化快速制作 Anki 记忆卡片**，一键导出 **Anki 原生可直接导入** 的文件格式，导入后无需手动配置，直接生成可用卡片，解决 Anki 官方制卡繁琐、效率低的问题。

---

## 二、核心功能需求
### 基础必选功能
1. **双模式制卡**
   - 单张制卡：可视化表单填写卡片正面、背面、标签、目标卡组
   - 批量制卡：支持多行文本粘贴（如单词表、知识点），自动拆分生成多张卡片
2. **卡片管理**
   - 实时预览卡片正反面效果
   - 卡片列表展示、单张编辑/删除、一键清空
   - 浏览器本地存储（刷新页面不丢失数据）
3. **一键导出**
   - 导出 **Anki 直接导入** 的标准格式（优先 CSV，兼容 APKG）
   - 自动处理中文编码，杜绝乱码
4. **极简操作**
   - 单页面设计，无学习成本，全年龄段可用

### 可选扩展功能
- 富文本编辑（加粗、换行、简单排版）
- 自定义卡片模板
- 响应式适配（手机/平板/电脑通用）

---

## 三、关键技术标准：Anki 可直接导入的格式规范
### 格式1：UTF-8 编码 CSV 文件（推荐首选）
#### 1. 格式规则（强制要求）
1. **编码**：必须为 **UTF-8**（解决中文乱码唯一方案）
2. **分隔符**：英文逗号 `,`（Anki 默认支持，无需修改设置）
3. **字段定义**（固定顺序，不可更改）：

| 字段名 | 必要性 | 说明 |
|--------|--------|------|
| 正面   | 必选   | 卡片提问内容（如：你好） |
| 背面   | 必选   | 卡片答案内容（如：Hello） |
| 标签   | 可选   | 多个标签用 **空格/分号** 分隔（如：英语 单词） |
| 卡组   | 可选   | 导入后自动归属的卡组名（如：英语词汇） |

4. **特殊处理**：字段包含逗号时，用英文双引号 `"` 包裹内容

#### 2. 标准文件示例
```csv
正面,背面,标签,卡组
你好,Hello,英语 单词,英语基础词汇
苹果,Apple,英语 水果,英语基础词汇
计算机,Computer,IT 术语,编程词汇
```
#### 3. Anki 导入效果
用户打开 Anki → 文件 → 导入 → 选择导出的 CSV → **自动识别字段、自动匹配卡组、自动加载标签**，一键完成导入。

### 格式2：APKG 原生包（进阶，无配置导入）
1. 定义：Anki 官方专有压缩包格式，包含卡片、模板、卡组全套数据
2. 实现方式：前端使用开源库 `anki-apkg-export` 直接生成
3. 优势：用户导入时**无需任何设置**，双击即可导入 Anki

## 四、技术选型（纯前端，零后端依赖）
| 技术/库 | 用途 |
|--------|------|
| HTML5 + CSS3 + JavaScript | 核心页面与逻辑 |
| Tailwind CSS | 快速搭建响应式界面 |
| localStorage | 浏览器本地存储卡片数据 |
| PapaParse | CSV 文件生成与下载 |
| anki-apkg-export | APKG 原生包生成 |

---

## 五、项目页面结构（单页应用）
1. **顶部导航**：项目名称、清空数据、帮助说明
2. **左侧制卡区**
   - 单张制卡表单（正面/背面/标签/卡组输入框）
   - 批量制卡文本域（粘贴多行内容自动拆分）
3. **中间预览区**：实时展示卡片正反面效果
4. **右侧列表区**：所有制作的卡片列表（编辑/删除按钮）
5. **底部导出区**：导出 CSV / 导出 APKG 按钮

---

## 六、核心数据流程
1. 用户输入卡片信息 → 前端校验（正面/背面非空）
2. 数据存入 `localStorage` → 实时渲染卡片列表
3. 点击导出按钮 → 按 **Anki CSV 标准** 生成文件
4. 浏览器自动下载文件 → 用户直接导入 Anki

---

## 七、开发强制要求
1. **导出文件必须 100% 兼容 Anki**：不允许出现编码错误、字段错位、中文乱码
2. **纯前端运行**：无需后端接口、无需数据库、无需登录
3. **操作极简**：全程不超过 3 步完成「制卡→导出→导入」
4. **兼容性**：支持 Chrome/Edge/ Firefox 主流浏览器

---

## 八、项目交付成果
1. 单个 HTML 文件（可直接双击打开，无需部署）
2. 完整的卡片编辑、预览、管理功能
3. 一键导出 Anki 兼容 CSV 文件功能
4. 简洁美观的响应式界面

---

## 九、用户使用流程（最终效果）
1. 打开 Web 页面 → 填写卡片正面/背面
2. 点击添加 → 预览卡片
3. 点击「导出 CSV」→ 下载文件
4. 打开 Anki → 导入文件 → 完成制卡

---

# 总结
1. 本项目核心是**Web 可视化制卡 + Anki 标准格式导出**，纯前端零门槛使用；
2. **CSV 是核心导出格式**，必须严格遵循 UTF-8 编码、固定字段、英文逗号分隔规则；
3. 最终交付一个开箱即用的 HTML 文件，用户无需任何技术基础即可快速制作 Anki 卡片。
